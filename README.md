## Features

| Feature | Description |
| --- | --- |
| **Dashboard** | Service status, live traffic sparklines, core version |
| **Subscriptions** | Add, update, activate, and delete Mihomo YAML subscriptions |
| **Nodes** | Group selector, latency sort, single / batch delay tests |
| **Proxy** | System proxy (GNOME + shell env) and Mihomo mixed-port settings |
| **Network modes** | **System** (proxy) and **TUN** (full-tunnel) with mutual exclusion |
| **Interfaces** | Textual TUI · Web UI on `:7887` · CLI for automation |

Runtime data stays under `.runtime/` by default — isolated from Clash Verge and other local clients.

## Quick Start

```bash
git clone <your-repo-url> ClashTX
cd ClashTX

./scripts/install.sh
sudo ./clashtx.sh help
sudo ./clashtx.sh start
sudo ./clashtx.sh          # open the TUI
sudo ./clashtx.sh ui       # open the Web UI at http://127.0.0.1:7887
```

The installer creates `.venv`, installs dependencies, and generates `clashtx.sh` at the repository root.

---

## Terminal TUI

Start the terminal interface with:

```bash
sudo ./clashtx.sh
```

The TUI is built for keyboard-first server workflows. Use it to monitor service status and traffic, manage subscriptions, switch proxy groups, run latency tests, and toggle System / TUN network modes.

![ClashTX Terminal TUI](asset/img-terminal.png)

Common shortcuts:

- `r` refreshes dashboard, proxy, and node data.
- `s` starts the Mihomo core.
- `x` stops the Mihomo core.
- `q` quits the TUI.

---

## Web UI

Start the browser interface with:

```bash
sudo ./clashtx.sh ui
```

Then open [http://127.0.0.1:7887](http://127.0.0.1:7887). Use `--host` and `--port` when you want to expose it on another address:

```bash
sudo ./clashtx.sh ui --host 0.0.0.0 --port 7887
```

The Web UI provides the same core controls as the TUI, with bilingual EN / 简体中文 switching, dark / light mode, subscription management, proxy mode controls, node testing, and live traffic charts.

![ClashTX Web UI](asset/img-webui.png)

## Commands

| Command | Description |
| --- | --- |
| `sudo ./clashtx.sh` | Start the Textual TUI |
| `sudo ./clashtx.sh ui [--host HOST] [--port PORT]` | Start the Web UI (default `:7887`) |
| `sudo ./clashtx.sh start` | Start the Mihomo core |
| `sudo ./clashtx.sh stop` | Stop the Mihomo core |
| `sudo ./clashtx.sh restart` | Restart the Mihomo core |
| `sudo ./clashtx.sh status` | Show service status |
| `sudo ./clashtx.sh mode system` | System proxy mode (disables TUN, enables proxy) |
| `sudo ./clashtx.sh mode tun` | TUN mode (disables system proxy) |
| `sudo ./clashtx.sh help` | Show help |

## Network Modes

### System Mode

- Disables TUN and regenerates `mihomo.yaml`
- Automatically enables system proxy (GNOME + `proxy.env`)
- Best for apps that respect OS / shell proxy settings

### TUN Mode

- Disables system proxy and enables Mihomo TUN in config
- Requires `/dev/net/tun` and **CAP_NET_ADMIN** on the Mihomo binary

One-time setup:

```bash
./vendor/tun/grant-caps.sh
# or
sudo setcap cap_net_admin,cap_net_bind_service+ep vendor/mihomo/verge-mihomo
```

Re-run after replacing the Mihomo binary. TUN helpers live in `vendor/tun/`.

## Configuration

Override runtime directories with environment variables:

| Variable | Purpose |
| --- | --- |
| `CLASHTX_CONFIG_DIR` | Config and subscriptions |
| `CLASHTX_DATA_DIR` | Core data |
| `CLASHTX_CACHE_DIR` | Logs and pid files |
| `CLASHTX_CORE_PATH` | Custom Mihomo binary path |
| `CLASHTX_TUN_DIR` | Custom TUN tools directory |
